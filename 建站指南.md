## 5. 腾讯云 COS 配置

### 5.1 创建存储桶

1. 登录 [腾讯云控制台](https://console.cloud.tencent.com/)
2. 搜索「对象存储」→ 进入 COS 控制台
3. 点击「创建存储桶」
4. 配置如下：

| 配置项 | 值 | 说明 |
|--------|-----|------|
| 名称 | `blog-hk` | 自定义名称 |
| 地域 | 中国香港 | 免备案，即开即用 |
| 访问权限 | 公有读私有写 | 允许公开访问博客内容 |

### 5.2 开启静态网站

1. 进入存储桶 → 左侧「基础配置」→「静态网站」
2. 点击「开启」
3. 配置：

| 配置项 | 值 |
|--------|-----|
| 索引文档 | `index.html` |
| 强制 HTTPS | 开启（需先配置证书） |

4. 记录「访问节点」地址，格式类似：
   ```
   https://blog-hk-xxxxx.cos-website.ap-hongkong.myqcloud.com
   ```

### 5.3 子账号与权限配置

为了安全，创建专用子账号进行部署：

1. 腾讯云控制台 → 搜索「访问管理」→ 用户 → 新建用户
2. 用户类型：**可访问资源并接收消息**
3. 用户名：`hexo-cos-deploy`
4. 创建完成后，记录 **SecretId** 和 **SecretKey**
5. 给子账号关联策略：**QcloudCOSFullAccess**（对象存储全读写权限）

## 6. 域名与 DNS 配置

### 6.1 域名规划

| 用途 | 域名 | 解析目标 |
|------|------|----------|
| 群晖 NAS 外网访问 | `billnas.cn` | 家庭公网 IP（A 记录） |

| 主机记录 | 记录类型 | 记录值 |
|----------|----------|--------|
| `blog` | `CNAME` | `blog-hk-xxxxx.cos-website.ap-hongkong.myqcloud.com` |

> 如果域名托管在 DNSPod，COS 控制台会提供「一键解析」按钮，点击即可自动配置。

### 7.3 开启强制 HTTPS

1. COS 控制台 → 存储桶 →「基础配置」→「静态网站」

## 8. GitHub Actions 自动部署

### 8.1 创建 GitHub 仓库

1. 在 GitHub 创建仓库（如 `blog`）
2. 将本地博客项目推送到仓库：

```bash
cd blog
git init
git add .
git commit -m '初始化博客'
git remote add origin https://github.com/用户名/仓库名.git
git push -u origin main
```

### 8.2 配置 GitHub Secrets

在 GitHub 仓库 → Settings → Secrets and variables → Actions，添加：

| Secret 名称 | 值 |
|--------------|-----|
| `COS_SECRET_ID` | 腾讯云子账号的 SecretId |
| `COS_SECRET_KEY` | 腾讯云子账号的 SecretKey |

### 8.3 创建部署工作流

创建文件 `.github/workflows/deploy.yml`：

```yaml
name: Deploy Hexo to Tencent Cloud COS

on:
  push:
    branches:
      - main

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - name: Checkout code
        uses: actions/checkout@v4
        with:
          submodules: true

      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: '20'

      - name: Install dependencies
        run: npm install

      - name: Generate static files
        run: npx hexo generate

      - name: Configure coscmd
        run: |
          pip install coscmd
          coscmd config -a ${{ secrets.COS_SECRET_ID }} -s ${{ secrets.COS_SECRET_KEY }} -b blog-hk-xxxxx -r ap-hongkong

      - name: Deploy to COS
        run: |
          cd ./public
          coscmd upload -rfs --delete ./ /
```

> ⚠️ 将 `blog-hk-xxxxx` 替换为你的实际存储桶名称。

### 8.4 推送触发部署

```bash
git add .
git commit -m '部署博客'
git push origin main
```

在 GitHub 仓库的 Actions 标签页可以查看部署进度。

## 10. 安全注意事项

### 10.1 密钥管理

- **永远不要**将 API 密钥、Token 等敏感信息提交到代码仓库
- 使用 GitHub Secrets 存储部署凭证
- 定期轮换密钥（建议每 3 个月）

### 10.2 子账号权限

- 部署使用专用子账号，不要用主账号密钥
- 子账号仅授予必要的最小权限（COS 读写）

### 10.3 证书管理

- 免费证书有效期 3 个月，到期前及时续期
- 可设置日历提醒

## 11. 常见问题

### Q: 部署成功但访问 404？

检查 COS 存储桶中的文件路径：
- 正确：`/index.html`（文件在根目录）
- 错误：`/public/index.html`（文件在 public 子目录）

确保部署脚本中 `cd ./public` 后再上传。

### Q: 访问显示不安全（HTTP）？

1. 确认已申请并绑定 SSL 证书
2. 确认已开启「强制 HTTPS」
3. 浏览器强制刷新：`Ctrl + Shift + R`

### Q: GitHub Actions 报错 AccessDenied？

子账号权限不足，需要关联 `QcloudCOSFullAccess` 策略。

### Q: 如何回滚到之前版本？

```bash
git log --oneline          # 查看提交历史
git revert HEAD            # 回滚最近一次提交
git push origin main       # 推送回滚
```
